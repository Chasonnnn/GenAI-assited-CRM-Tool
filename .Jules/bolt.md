## 2024-05-24 - Efficient Aggregate Queries with Conditional Sums
**Learning:** In the `_ensure_empty_org` function in `admin_import_service.py`, 20 separate `db.query(Model).filter(...).count()` database queries were being executed sequentially. Each one creates an unnecessary `SELECT count(*) FROM (SELECT ...)` subquery. Furthermore, combining counts via `func.count(Model.id)` is faster but still requires round-trips. Conditional aggregation (`func.coalesce(func.sum(case((condition, 1), else_=0)), 0)`) can condense multiple counts on the same table, but since these are counts on 20 *different* tables, conditional aggregation isn't applicable. Instead, using `db.scalar(select(func.count(Model.id)).where(...))` optimizes each count by performing a direct index aggregate without generating inefficient subqueries.
**Action:** Replace `db.query(Model).filter(...).count()` with `db.scalar(select(func.count(Model.id)).where(...))` to minimize database CPU load and speed up counts.
