## 2025-03-08 - Optimized aggregate COUNT query subquery generation in SQLAlchemy
**Learning:** In SQLAlchemy, calling `.count()` directly on a query object (e.g., `db.query(Model).filter(...).count()`) generates inefficient subqueries (like `SELECT count(*) FROM (SELECT ...)`) because it tries to account for potential result set grouping/ordering. When optimizing for basic count calculations (like checking if limits are exceeded), it is significantly faster and less resource-intensive to use `db.scalar(select(func.count(Model.id)).where(...))` to execute a direct aggregate count query.
**Action:** Always favor `db.scalar(select(func.count(Model.id)).where(...))` over `.count()` for single aggregate count checks without complex groupings, as it drastically reduces the SQL complexity and query overhead on the PostgreSQL backend.
