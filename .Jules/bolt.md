## 2024-05-24 - Efficient Aggregate Queries with Conditional Sums
**Learning:** In the `_ensure_empty_org` function in `admin_import_service.py`, using `db.scalar(select(func.count(Model.id)).where(...))` optimizes each count by performing a direct index aggregate without generating inefficient subqueries. However, some models like `UserNotificationSettings` have a composite primary key or a different primary key column (e.g. `user_id`) and don't have an `id` attribute. Using `func.count(Model.id)` blindly causes an AttributeError. Instead, use `func.count()` with no arguments (which translates to `COUNT(*)`) or use the correct primary key column.
**Action:** Replace `func.count(Model.id)` with `func.count()` or use the model's actual primary key to safely handle models without an `id` column.
