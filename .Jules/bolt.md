## 2024-05-18 - Replacing Inefficient .count() in SQLAlchemy\n**Learning:** In SQLAlchemy, calling `.count()` directly on a query (e.g., `db.query(Model).filter(...).count()`) triggers an inefficient subquery that wraps the entire original SQL query (`SELECT count(*) FROM (SELECT ...)`). Using `db.scalar(select(func.count(Model.id)).where(...))` compiles into a direct aggregate function call (`SELECT COUNT(id) FROM ...`), avoiding subquery overhead and improving DB execution time.\n**Action:** Use `select(func.count(Model.id))` to calculate aggregate counts across the codebase, avoiding the `db.query(Model).count()` approach.
