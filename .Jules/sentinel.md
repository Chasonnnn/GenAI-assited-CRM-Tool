## 2025-03-09 - CSV Injection Prevention\n**Vulnerability:** Uploaded CSV files were not sanitized, allowing formula injection attacks (CSV Injection) when opened in spreadsheet software.\n**Learning:** When writing security sanitization functions (like `sanitize_csv`), it is critical to use robust parsing libraries (like `csv.reader`) rather than naive string replacement to avoid corrupting data. Furthermore, security functions must fail-closed (raise exceptions instead of passing through data) on invalid inputs.\n**Prevention:** Apply `sanitize_csv` across all file upload points (e.g. `attachment_service.py` and `form_submission_service.py`) for relevant MIME types (`text/csv`, `application/csv`, `application/vnd.ms-excel`) and enforce a fail-closed paradigm.\n
