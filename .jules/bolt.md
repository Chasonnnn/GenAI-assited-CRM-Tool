## $(date +%Y-%m-%d) - Optimize SQLAlchemy Pagination Counts
**Learning:** Found a recurring performance anti-pattern across the backend codebase: `query.count()` is being executed unconditionally on paginated lists. When paginating with `.limit()` and `.offset()`, if the length of the fetched result list is strictly less than the limit, we can deduce we are on the final page (or only page). In this scenario, we can mathematically calculate the total as `offset + len(results)` (provided `offset == 0` or `len(results) > 0`). This entirely avoids executing the separate, potentially expensive `.count()` query against the database for the last page or small data sets.
**Action:** When working with SQLAlchemy pagination abstractions, execute the `items = query.offset().limit().all()` query first. Then conditionally calculate the total count based on the number of items fetched, falling back to `query.count()` only if the fetched items hit the limit (indicating there may be more pages) or if zero items are returned past page 1.
