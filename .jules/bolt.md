## 2026-06-09 - Combine count queries using conditional aggregation
**Learning:** Performing multiple independent `.count()` queries on the same table with overlapping base conditions introduces significant latency due to redundant DB roundtrips. When rewriting them into a single aggregate query using `func.sum(case(...))`, ensure query-specific filters are strictly scoped to the conditional blocks and never leak into the base query `.filter()`. Also, defensively wrap object lookups (e.g. `snapshot = config.get("snapshot")`) and check attributes (`hasattr(snapshot, "stages")`) to handle malformed configs or missing data correctly without causing `AttributeError`s.
**Action:** Use conditional aggregation (`func.coalesce(func.sum(case((cond, 1), else_=0)), 0)`) to combine identical base count metrics. Ensure dictionary access handles `None` safely.
