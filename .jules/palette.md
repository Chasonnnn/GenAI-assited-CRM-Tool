## 2024-04-10 - Screen Reader Only Text in Icon Buttons
**Learning:** Found a persistent anti-pattern where icon-only components (like `<Button>`) use nested `<span className="sr-only">Label</span>` for accessible names. This practice is discouraged because wrapping custom interactive UI components with nested visually-hidden DOM nodes can complicate focus outlines and create inconsistent semantic experiences across different browser+screen-reader combinations compared to using the native `aria-label` attribute directly on the parent button.
**Action:** When adding accessible names to icon-only buttons or interactive elements, apply the `aria-label` attribute directly to the parent interactive element (e.g., `<Button aria-label="Action">`) and mark the inner icon element as decorative with `aria-hidden="true"`. Do not use a nested `<span className="sr-only">`.
