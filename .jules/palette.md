## 2024-05-15 - Calendar Navigation Accessibility
**Learning:** Dynamic text that updates on user interaction (like a calendar's current month/date display) is completely silent to screen readers unless explicitly marked up. Users clicking "Next" or "Previous" buttons on a calendar without `aria-live` would hear the button's action but receive no confirmation of what the new date range is.
**Action:** When adding `aria-label` to icon-only navigation controls (like prev/next buttons), ALWAYS ensure the accompanying dynamic content display (like the current month/year text) has `aria-live="polite"`. This ensures the screen reader announces the new context after the navigation action completes. Also, always add `aria-hidden="true"` to the decorative inner SVG icons to prevent redundant noise.
